## 遇到的问题与解决方案

### 测试结果

- **测试执行**：`mvn test`
- **结果**：全部通过
- **报告输出**：`backend/back/target/surefire-reports/`

- **问题 1：运行测试时 Spring Boot 上下文会尝试连接本机 MySQL**
  - **现象**：`@SpringBootTest` 的 context load 测试会启动完整应用并读取 `application.yml`，如果本机未配置数据库则可能失败或卡住
  - **解决**：将该类调整为不启动 Spring 上下文的空测试（只保留 JUnit5 测试方法），将重点放在 Service 的 Mockito 单测和 Controller 的 WebMvc 切片测试
- **问题 2：Shop 兑换成功用例中订单 ID 断言失败**
  - **原因**：业务里 `shopRepository.save(order)` 后直接返回 `order.getId()`，单测需要让 mock 的 `save` 行为把 ID 回写到入参对象
  - **解决**：对 `shopRepository.save(...)` 使用 `thenAnswer`，在保存时把 `id` 设置到传入的 `order` 上再返回

## 心得体会

实际开发中需要通过单元测试验证正确性。Mock 机制使得能在不依赖真实数据库和复杂网络环境的情况下，精准测试业务逻辑的严密性。覆盖率是衡量测试全面性的客观标准，业务逻辑中覆盖率越高越好。自动化测试是养成良好工程习惯、提高代码可维护性的核心手段。目前项目逻辑尚未全部完成，待后续后端代码完成后将进行进一步测试。
