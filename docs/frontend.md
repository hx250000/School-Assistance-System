# Frontend Design - Android Client

## 1. 前端概述

本项目的前端采用 **Android 原生开发**，主要负责用户界面展示、用户交互以及与后端服务进行通信。

Android 客户端通过 HTTP 请求与 SpringBoot 后端进行数据交互，实现任务发布、任务抢单、积分查看、商城兑换等核心功能。

前端采用 **MVVM 架构模式**，以提高代码的可维护性和可扩展性。

---

## 2. 技术栈

本项目 Android 前端主要使用以下技术：

- **Kotlin**：主要开发语言
- **MVVM 架构**：提升代码结构清晰度
- **Retrofit**：网络请求框架
- **RecyclerView**：列表数据展示
- **Room**：本地数据库缓存
- **Material Design**：UI 设计规范
- **WebSocket（可扩展）**：实时聊天功能

---

## 3. 前端整体架构

Android 客户端采用 **MVVM 架构**。

架构结构如下：
View (Activity / Fragment)
│
│
ViewModel
│
│
Repository
│
│
Network (Retrofit API)


各层职责：

### View 层

负责 UI 展示和用户交互，例如：

- Activity
- Fragment
- RecyclerView

### ViewModel 层

负责处理 UI 逻辑，例如：

- 获取任务列表
- 发布任务
- 抢任务
- 更新界面数据

### Repository 层

负责数据来源管理，例如：

- 网络请求
- 本地缓存
- 数据整合

本项目主要包含以下页面：

### 4.1 首页（任务列表）

功能：

- 显示所有可接取任务
- 按任务类型分类
- 点击查看任务详情

主要组件：

- RecyclerView
- CardView
- 下拉刷新

显示信息：

- 任务标题
- 任务类型
- 所需人数
- 积分奖励
- 发布时间

---

### 4.2 发布任务页面

用户可以创建新任务。

填写内容包括：

- 任务标题
- 任务描述
- 任务类型
- 需要人数
- 积分奖励
- 截止时间

提交后通过 API 发送到后端服务器。

---

### 4.3 任务详情页面

显示任务详细信息：

- 任务标题
- 发布者
- 描述
- 参与人数
- 积分奖励
- 截止时间

用户可以点击 **抢任务**。

---

### 4.4 我的任务页面

用户可以查看：

- 已发布任务
- 已参与任务
- 进行中的任务
- 已完成任务

---

### 4.5 积分商城页面

用户可以浏览商城商品。

功能包括：

- 查看商品列表
- 查看商品详情
- 使用积分兑换商品

---

### 4.6 用户中心页面

显示用户信息：

- 用户昵称
- 当前积分
- 信用评分
- 历史任务记录

## 5. 网络通信设计

Android 客户端通过 **RESTful API** 与后端进行通信。

使用 **Retrofit** 实现网络请求。

示例 API：

### 获取任务列表
GET /api/tasks


返回数据：


[
{
"id": 1,
"title": "帮带一份晚饭",
"type": "生活",
"rewardPoints": 10,
"currentPeople": 1,
"maxPeople": 2
}
]


---

## 6. 本地缓存设计

为了减少网络请求和提升用户体验，可以使用 **Room 数据库**缓存部分数据，例如：

- 最近浏览的任务
- 用户信息
- 任务历史记录

缓存策略：


优先读取本地缓存
若缓存不存在 → 请求服务器
更新本地数据
